# Submission Guide

## Overview

This guide explains how to prepare and submit the Redrob Candidate Ranking System for the hackathon.

## Submission Requirements

### Required Deliverables

1. **GitHub Repository**
   - Complete source code
   - Documentation
   - Sample datasets
   - Deployment configuration

2. **Streamlit Sandbox**
   - Deployed and accessible
   - Functional demo
   - Sample data included

3. **Submission CSV**
   - Generated by the system
   - Contains Top 100 candidates
   - Follows specified format

4. **Documentation**
   - README.md
   - Architecture diagram
   - Setup instructions
   - Usage guide

- [ ] GitHub repository is public

- [ ] Submission CSV follows format
- [ ] Runtime is <5 minutes
- [ ] Memory usage is reasonable

- [ ] No external API calls

## Step 2: Validate Submission

### Check CSV Format

```bash
# Using validation script
python unified/scripts/validate.py
```

### Manual Validation

```python
import csv

with open('unified/outputs/submissions/submission.csv', 'r') as f:
    reader = csv.DictReader(f)
    rows = list(reader)
    
    # Check required columns
    required_columns = ['candidate_id', 'rank', 'score', 'reasoning']
    assert all(col in reader.fieldnames for col in required_columns)
    
    # Check row count
    assert len(rows) <= 100
    
    # Check data types
    for row in rows:
        int(row['candidate_id'])
        int(row['rank'])
        float(row['score'])
        str(row['reasoning'])
    
    print("Validation passed")
```

### Expected Output Format

```csv
candidate_id,rank,score,reasoning
1001,1,1.6146,"Exceptional 7.5-year Senior Search Engineer with proven expertise in Elasticsearch, FAISS..."
1002,2,1.4532,"Highly recommended founding candidate with 5.0 years of experience..."
```

## Step 5: Deploy Streamlit Sandbox

### Deploy to Streamlit Cloud

1. Go to https://share.streamlit.io
2. Sign in with GitHub
3. Click "New app"
4. Select repository: `your-username/TASK15`
5. Select branch: `main`
6. Set main file path: `sandbox/app.py`
7. Click "Deploy"

### Verify Deployment

1. Open deployed app URL
2. Upload sample dataset
3. Generate rankings

## Troubleshooting

### Issue: "Submission CSV not found"
**Cause**: Submission not generated
**Solution**: Run `python unified/scripts/generate_submission.py`

### Issue: "Invalid CSV format"
**Cause**: CSV doesn't follow specification
**Solution**: Check CSV structure and data types

### Issue: "Runtime too long"
**Cause**: Large dataset or inefficient code
**Solution**: Optimize code or reduce dataset size

### Issue: "Sandbox not working"
**Cause**: Deployment issue
**Solution**: Check Streamlit Cloud logs and redeploy

### Issue: "Import errors"
**Cause**: Missing dependencies
**Solution**: Update requirements.txt and redeploy

## Best Practices

1. **Start early** - Don't wait until the last minute
2. **Test thoroughly** - Validate everything before submission
3. **Document well** - Clear documentation helps judges
4. **Keep it simple** - Focus on core functionality
